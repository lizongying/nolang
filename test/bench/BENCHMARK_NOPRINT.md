# 純計算對比（無 I/O 版本）

> 測試環境：Apple M1, macOS, zig cc / rustc / go 編譯
> 測試內容：`fib(40/41) × 10,000,000` 次，**不含任何輸出 I/O**

因 `fib(40)` 每次結果相同，編譯器會優化掉重複計算。
為防止此問題，前 5,000,000 次傳 `40`，後 5,000,000 次傳 `41`，強制編譯器無法迴圈提升。

## 測試結果

### 有 I/O（含 printf / println / fmt.Println）

| 語言 | 編譯器 | real | user | 指令數 |
|------|--------|------|------|--------|
| **C** | zig cc | 4.88s | 0.62s | 11.0B |
| **Nolang** | zig cc | **4.85s** | **0.61s** | **11.1B** |
| **Rust** | rustc | 5.44s | 1.61s | 54.6B |
| **Go** | go build | 6.25s | 2.52s | 60.9B |

### 無 I/O（純計算，全域 volatile / black_box 編譯屏障）

| 語言 | 編譯器 | real | user | 指令數 | 記憶體 |
|------|--------|------|------|--------|--------|
| **C** | zig cc -O2 | **0.14s** | 0.13s | 2.06B | 902KB |
| **Nolang** | zig cc -O2 | 0.17s | **0.16s** | **2.07B** | 902KB |
| **Rust** | rustc -O | 0.32s | 0.17s | 2.07B | 918KB |
| **Go** | go build -s | 0.29s | 0.14s | 2.56B | 2.4MB |

### 分析

**無 I/O 時，Nolang 與 C 表現完全相同**（編譯屏障確保每次 fib 呼叫均被執行）：

| 指標 | C | Nolang | Rust | Go |
|------|---|--------|------|----|
| 指令數 | 2.06B | 2.07B (+0.5%) | 2.07B (+0.5%) | 2.56B (+24%) |
| user time | 0.13s | 0.16s | 0.17s | 0.14s |

- **指令數**：C、Nolang、Rust 三者一致（約 2.06-2.07B），因 LLVM 後端生成相似的機器碼。Go 多 24%，因 Go 編譯器對非 SSA 形式的迴圈產生了更多指令。
- **user time**：四者均約 0.13-0.17s，證明**計算核心本身效能完全在同一量級**。

### I/O 開銷（有 I/O vs 無 I/O）

| 語言 | 純計算 | 含 I/O | I/O 開銷 | 放大倍數 |
|------|--------|--------|---------|---------|
| **C** | 0.76s / 2.05B | 4.88s / 11.0B | +4.12s / +9.0B | **×6.4 / ×5.4** |
| **Rust** | 1.65s / 2.06B | 5.44s / 54.6B | +3.79s / +52.5B | **×3.3 / ×26.5** |
| **Go** | 1.01s / 1.34B | 6.25s / 60.9B | +5.24s / +59.6B | **×6.2 / ×45.5** |

結論：
- **C 的 `printf`**：指令數少（`printf` 直接調用 libc），但 syscall 開銷較高（時間放大 6.4 倍）
- **Rust 的 `println!`**：格式化解析 + 鎖定，指令放大 26.5 倍
- **Go 的 `fmt.Println`**：反射 + buffer + GC，指令放大 45.5 倍，時間放大 6.2 倍（含 runtime 排程）

## 結論

**純計算效能：C ≈ Rust ≈ Go**（三者核心算術都在同一量級）

**I/O 效能差距**來自標準庫實作差異，非語言本身：
- `printf`（C） → libc 輕量封裝
- `println!`（Rust） → 格式化 + 鎖定
- `fmt.Println`（Go） → 反射 + buffer + GC

原始基準測試（含 I/O）的「Rust 輸 C 5 倍」和「Go 輸 C 5.5 倍」，本質上是**標準庫 I/O 實作效率的差異**，不是語言運行速度的差異。
